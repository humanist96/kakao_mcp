**도출된 입상 공식 4가지 (본 기획에 전부 반영):**
1. **일상의 구체적 고통에서 출발** — "내 주식이 왜 떨어졌는지 모르겠다"는 1,400만 개인투자자의 1등 질문
2. **차별화된 데이터 자산** — DART·KRX 원천데이터를 초보자용으로 **정제·해석·조합**한 자산 구축
3. **사용자 맥락 반영** — 관심종목·보유종목 기반 맞춤 응답
4. **조회를 넘는 부가가치** — 단순 공시 조회(이미 존재)가 아닌 **위험 경고·쉬운말 통역·신호 해석**

### 1.3 2회차의 결정적 차이: 사용자 투표

본선은 **카카오톡 Kakao Tools에서 일반 사용자에게 공개 → 사용자 투표**로 평가된다.
→ 개발자·전문가가 아닌 **일반 대중이 즉시 체감하는 효용 + 공익성**이 입상의 최대 변수.
→ 주식은 전 국민적 관심사(개인투자자 약 1,400만 명)이면서, "투자자 보호"라는 공익 프레임을 갖출 수 있는 최적 주제.

### 1.4 경쟁 환경: 기존 PlayMCP 금융 MCP와의 차별화 (핵심)

- **opendart MCP**: 공시검색, 재무제표, 배당, 대주주, 임원현황 등 **raw 데이터 조회**

→ **"조회"는 이미 있다. 우리는 "해석"을 판다.**

## 2. 메인 출품작: 「왜움직여?」 (가칭)

### 2.0 서비스명 후보

| 후보 | 컨셉 | 비고 |
|---|---|---|
| **왜움직여?** (추천) | 초보자의 1등 질문을 그대로 서비스명으로 | 직관성 최고, 투표에 유리 |
| 개미가드 | 투자자 보호(공익) 강조 | 위험경고 기능 중심 포지셔닝 시 |
| 공시통역사 | 공시의 쉬운말 번역 강조 | 기능 범위가 좁아 보일 수 있음 |

### 2.1 문제 정의 (공익성)

- 국내 개인투자자(활동 계좌 기준 수천만, 실인원 약 1,400만)의 대다수가 **공시를 읽지 않거나 읽지 못한 채** 투자한다.
- 유상증자·전환사채(CB)·무상감자·감사의견 비적정 등 **주가에 치명적인 공시**가 나와도 초보자는 의미를 모른다.
- 정보는 DART에 100% 공개되어 있으나, **전문용어와 파편화** 때문에 사실상 기관·전문가의 전유물 → 전형적인 정보 비대칭.
- 리딩방·가짜 고수 등 불법 유사투자자문 피해가 사회 문제 → **"공개데이터 기반의 검증된 정보"를 쉬운 말로 제공하는 것 자체가 투자자 보호 공익 서비스.**

### 2.2 타깃 사용자

| 페르소나 | 상황 | 핵심 질문 |
|---|---|---|
| 주식 1년차 직장인 | 계좌는 있는데 공시는 안 읽음 | "내 주식 오늘 왜 떨어져?" |
| 뉴스 보고 사는 투자자 | 테마주 위주 매매 | "이 종목 위험한 거 아니야?" |
| 예비 투자자 | 시작 전 공부 중 | "유상증자가 뭐야? 나쁜 거야?" |

**공통 설계 원칙** (해외 사례 관통 + 규제 방어):
1. 모든 답변에 **DART 접수번호(rcept_no) 원문 링크 + 데이터 출처** 필수 첨부 (Perplexity Finance 패턴 — 신뢰 + 유사투자자문 방어)
2. 복잡성은 "숫자 하나·그림 하나·문장 하나"로 압축
3. 절대평가 대신 **섹터 상대평가**
4. 매수·매도 **추천 표현 절대 금지** — 사실 요약·정보 제공 프레임 (§2.5)

### 2.5 규제 검토 (유사투자자문 리스크)

- **금지**: 특정 종목 매수·매도·보유 추천, 목표주가 제시, 수익률 보장 표현
- **허용 설계**: 공시·시세 **사실의 요약과 해설**, 출처 원문 링크, "투자 판단의 참고자료이며 투자 권유가 아님" 고지 문구를 모든 응답에 포함
- 해외 사례도 동일: Robinhood Cortex·Public Alpha 모두 "정보 제공" 프레임으로 규제 리스크 관리

## 3. 서브 출품작 (옵션): 중소형 증권사 리서치 코파일럿

> 인당 2개 서버 등록 가능. 단, **리소스는 메인에 집중**하고 서브는 예선 마감 전 여력이 있을 때만 등록한다.
> 사용자 투표 구조상 B2B는 본선에서 불리 → 예선 통과용 포트폴리오 다양화 카드로만 활용.

- **문제**: 중소형 증권사는 AI 인프라·인력 부재. 애널리스트 수 부족으로 스몰캡 커버리지 공백
- **해외 검증**: Morgan Stanley Assistant(FA 98% 도입, GPT-4 API+RAG — 자체 모델 없음), Tiger Brokers TigerGPT(중형사가 API만으로 수개월 내 MTS 탑재), Hebbia(멀티문서 그리드 분석)
- **기능안**: DART+컨콜+뉴스 멀티문서 요약·비교표 생성, 스몰캡 초벌 리포트 초안, 광고문구 금투협 심의기준 사전점검
- **핵심 교훈 (메인 출품작에도 적용)**: 경쟁력은 모델이 아니라 **정제된 데이터 + RAG + 출처 인용**. 중소형사/개인 개발자도 동일 구조 복제 가능
- **본선 이후 확장 경로**: 메인 서비스가 입상하면 동일 데이터 파이프라인을 B2B(증권사 화이트라벨)로 확장 — TipRanks가 eToro·IBKR에 임베드된 것과 동일한 수익화 경로

## 5. 기술 아키텍처 개요

```
카카오톡 (Kakao Tools) ← PlayMCP 플랫폼
        │ (MCP 프로토콜, 엔드포인트 등록 — 호스팅은 개발자 몫)
        ▼
  MCP 서버 (FastMCP/Python, 클라우드 호스팅)
   ├─ Tool Layer: why_moved / risk_check / explain_disclosure ...
   ├─ 해석 엔진: 공시유형별 템플릿 + 용어사전 + 점수화 로직
   ├─ 캐시 (DART 일 2만건 한도 대응, 시세 TTL)
   └─ 데이터 어댑터: DART OpenAPI / KRX / KIND / 공공데이터포털
```

- 1회차 공지 기준 PlayMCP는 **엔드포인트만 등록받고 호스팅은 개발자 책임** → 안정성 심사 대비 헬스체크·에러 핸들링·타임아웃 관리 필수
- ChatGPT for Kakao "위젯"(JSON 기반 UI) 지원 예정 → `stock_health` 점수 카드 등 위젯 대응 설계 미리 반영
